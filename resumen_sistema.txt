DESCRIPCIÓN DEL SISTEMA
-----------------------
Sistema web completo para registro de ventas y gestión de eventos para 
negocio de tortas y banquetería (dulce y salada).

MÓDULOS IMPLEMENTADOS:
- Autenticación JWT con roles (ADMIN, SELLER, OPERATIONS)
- Clientes (CRUD completo)
- Productos/Servicios (CRUD con categorías)
- Ventas (CRUD con ítems dinámicos, estados, cálculos automáticos)
- Eventos (CRUD con documentos PDF)
- Dashboard (indicadores del mes)
- Auditoría (logs de cambios)

================================================================================

FASE 1: DISEÑO
--------------

ARQUITECTURA:
- Backend: NestJS con patrón modular (domain/application/infrastructure)
- Frontend: Next.js 14 con App Router
- Base de datos: PostgreSQL con Prisma ORM

MODELO DE DATOS (TABLAS):
- roles, users, clients, products
- sales, sale_items
- events, event_documents
- audit_log

ENDPOINTS API (REST /api/v1):
- Auth: POST /auth/login, GET /auth/me
- Clients: CRUD completo
- Products: CRUD completo
- Sales: CRUD + PATCH /:id/status
- Events: CRUD + PATCH /:id/status
- Documents: POST/GET/DELETE en /events/:id/documents
- Dashboard: GET /dashboard/summary

COMANDOS PARA EJECUTAR:
cd backend
npm install

# Desarrollo
cp .env.example .env
# Configurar DATABASE_URL_DEV en .env
npm run prisma:generate:dev
npm run prisma:migrate:dev
npm run prisma:seed:dev
npm run start:dev    # Servidor en http://localhost:3001

# Producción
npm run build
npm run start:prod   # Usa DATABASE_URL_PROD

FASE 4: VENTAS
--------------

BACKEND:
- Endpoints: GET/POST/PUT/PATCH/DELETE /api/v1/sales
- Cálculo automático de subtotal, descuento, total
- Validación: mínimo 1 ítem por venta
- Estados: PENDING, CONFIRMED, IN_PREPARATION, DELIVERED, CANCELLED

FRONTEND (/sales):
- Listado con filtros por estado
- Formulario con items dinámicos (agregar/eliminar)
- Selección de cliente y productos
- Cálculo automático en tiempo real
- Badges de estado con colores

================================================================================

FASE 5: EVENTOS
----------------

BACKEND:
- Endpoints: GET/POST/PUT/PATCH/DELETE /api/v1/events
- Tipos: CUMPLEANOS, MATRIMONIO, BAUTIZO, EMPRESA, OTRO
- Estados: QUOTED → CONFIRMED → IN_PREPARATION → EXECUTED → INVOICED → CLOSED
- Documentos: upload/download PDF (orden de compra, factura)
- Campos: fecha, hora, lugar, cantidad personas, servicio, observaciones

FRONTEND (/events):
- Listado con filtros por estado
- Formulario completo de evento
- Diálogo de gestión de documentos
- Subida de PDF con validación
- Descarga de documentos
